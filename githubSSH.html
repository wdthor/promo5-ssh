SSH

SSH (Secure Shell): est à la fois un programme informatique et un protocole de communication sécurisé.

Fonction: Session à distance sécurisée

Les clés SSH permettent de se connecter (faire un push) sur GitHub sans avoir à renseigner à chaque fois ses identifiants.

Générer des clés SSH

Avant de générer une clé SSH, on vérifie qu'elle n'existe pas déjà

Pour vérifier les clés SSH existant

1- Ouvrer le terminal et saisissez:

            
            ls -al ~/.ssh

2- Vérifier dans la liste si il existe déjà une clé SSH publique

Par défault, les nom de fichier des clés publiques sont:


            id_dsa.pub
            id_ecdsa.pub
            id_ed25519.pub

Avantages:

  • Elles garantissent l'authenticité, confidentialité et la l'integrité de ses données.
  • La transmission des données est chiffré, donc théoriquement peuvent etre difficilement comprise.

Faiblesses:

  • Si les clés privées ne peuvent pas être protégées, la sécurité n'est pas meilleure qu'avec l'authentification par mot de passe.

Conclusion

Globalement, on peut retenir de SSH est un moyen plus sécurisé pour envoyer des données à distance.
